# Project Internal Assessments & Schedule

## Project Timeline (Semester 6)

### Phase 1: Planning & Analysis (Weeks 1-2)
*   **Objective**: Define scope, requirements, and feasibility.
*   **Activities**:
    *   Requirement gathering with team and guide.
    *   Literature survey and Existing System Review.
    *   Finalizing the "Arivolam" concept and feature list.
    *   **Deliverable**: Abstract, System Requirements Specification (SRS).

### Phase 2: Design & Prototyping (Weeks 3-5)
*   **Objective**: Architect the system and design the UI/UX.
*   **Activities**:
    *   Database Schema Design (ER Diagrams).
    *   UI/UX Wireframing (Figma/Adobe XD).
    *   System Architecture (Next.js Front-end, Backend API structure).
    *   **Deliverable**: Design Document, UI Prototypes, Data Flow Diagrams (DFD).

### Phase 3: Development - Core Modules (Weeks 6-10)
*   **Objective**: Build the foundational features.
*   **Activities**:
    *   **Weeks 6-7**: Authentication (Admin/Teacher/Student), Dashboard Setup.
    *   **Weeks 8-9**: Academic Modules (Attendance, Assignments, Progress Reports).
    *   **Week 10**: Communication Modules (Notifications, Basic Group Chat).
    *   **Internal Assessment 1**: Presentation of partial implementation (Design + Core/Auth).

### Phase 4: Development - Advanced Features (Weeks 11-14)
*   **Objective**: Implement "Wow" factors and integration.
*   **Activities**:
    *   Campus Map & Navigation (AR Integration).
    *   AI Integration & Video Conferencing modules.
    *   Payment Gateway (Fees) and Library Systems.
    *   **Internal Assessment 2**: Demo of working prototype with advanced features.

### Phase 5: Testing & Refinement (Weeks 15-16)
*   **Objective**: Ensure stability and usability.
*   **Activities**:
    *   Unit Testing and Integration Testing.
    *   User Acceptance Testing (UAT) with peers.
    *   Code optimization and UI polishing (Animations, Themes).

### Phase 6: Final Documentation & Deployment (Weeks 17-18)
*   **Objective**: Final submission.
*   **Activities**:
    *   Preparation of Final Project Report.
    *   Deployment to hosting platform (Vercel/Netlify).
    *   **Final Presentation**: Comprehensive demo and viva voce.

---

## Internal Assessment Metrics
Evaluations will be based on:
1.  **Innovation**: Implementation of AR/Map and AI features.
2.  **Completeness**: Successful integration of all proposed modules.
3.  **UI/UX Quality**: Adherence to the "futuristic, sleek" design requirement.
4.  **Code Quality**: Modularity, cleanliness, and use of modern practices (Next.js).
